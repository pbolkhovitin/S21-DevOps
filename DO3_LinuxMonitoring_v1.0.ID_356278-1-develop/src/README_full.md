# Linux Monitoring v1.0 - Полное техническое описание

Документация по проекту Linux Monitoring, описывающая все компоненты, алгоритмы и взаимодействия.

## Структура проекта

```
src/
├── 01/               # Часть 1: Базовый скрипт
├── 02/               # Часть 2: Сбор системной информации
├── 03/               # Часть 3: Цветовой вывод
├── 04/               # Часть 4: Конфигурация цветов
├── 05/               # Часть 5: Анализ файловой системы
└── lib/              # Библиотеки функций
    ├── functions.sh  # Функции сбора системной информации
    ├── colors.sh     # Функции цветного вывода
    ├── config.sh     # Функции работы с конфигурацией
    └── analyzer.sh   # Функции анализа файловой системы
```

## Подробное описание компонентов

### src/01/main.sh - Базовый скрипт

**Назначение:**
Первый этап проекта, демонстрирующий базовую обработку аргументов командной строки.

**Алгоритм работы:**
1. Проверка наличия аргумента
2. Проверка, что аргумент не является числом
3. Вывод значения аргумента

**Аргументы:**
- `$1` - строка для проверки

**Пример использования:**
```bash
./src/01/main.sh "test"
```

**Ожидаемый вывод:**
```
test
```

### src/02/main.sh - Сбор системной информации

**Назначение:**
Сбор и вывод основной системной информации с возможностью сохранения в файл.

**Алгоритм работы:**
1. Проверка зависимостей через `check_dependencies()`
2. Сбор информации через функции из `functions.sh` с валидацией через `call_with_validation()`
3. Вывод информации через `print_system_info()`
4. Запрос на сохранение в файл через `prompt_yes_no()`
5. Сохранение в файл с именем в формате ДД_ММ_ГГ_ЧЧ_ММ_СС.status

**Используемые функции:**
- `call_with_validation("get_hostname")` - имя хоста
- `call_with_validation("get_timezone")` - часовой пояс
- `call_with_validation("get_user")` - текущий пользователь
- `call_with_validation("get_os")` - информация об ОС
- `call_with_validation("get_date")` - текущая дата и время
- `call_with_validation("get_uptime")` - время работы (читаемый формат)
- `call_with_validation("get_uptime_sec")` - время работы (в секундах)
- `call_with_validation("get_ip")` - IP-адрес
- `call_with_validation("get_mask")` - маска подсети
- `call_with_validation("get_gateway")` - шлюз
- `call_with_validation("get_ram_total")` - общая RAM
- `call_with_validation("get_ram_used")` - использованная RAM
- `call_with_validation("get_ram_free")` - свободная RAM
- `call_with_validation("get_space_root")` - общий объем корневого раздела
- `call_with_validation("get_space_root_used")` - использованный объем корневого раздела
- `call_with_validation("get_space_root_free")` - свободный объем корневого раздела

**Пример использования:**
```bash
./src/02/main.sh
```

### src/03/main.sh - Цветовой вывод

**Назначение:**
Визуальное оформление вывода с настройкой цветовой схемы через параметры командной строки.

**Алгоритм работы:**
1. Проверка аргументов (4 числовых параметра)
2. Валидация значений цветов через `validate_all_colors()`
3. Проверка конфликтов цветов через `check_all_color_conflict()`
4. Сбор информации через `call_with_validation()`
5. Вывод информации через `print_colored_system_info()`

**Аргументы:**
- `$1` - цвет фона для названий (1-6)
- `$2` - цвет текста для названий (1-6)
- `$3` - цвет фона для значений (1-6)
- `$4` - цвет текста для значений (1-6)

**Цвета:**
- 1 - white
- 2 - red
- 3 - green
- 4 - blue
- 5 - purple
- 6 - black

**Пример использования:**
```bash
./src/03/main.sh 1 2 3 4
```

### src/04/main.sh - Конфигурация цветов

**Назначение:**
Реализация чтения цветовой схемы из конфигурационного файла.

**Алгоритм работы:**
1. Проверка аргумента `--default`
2. Чтение конфигурации из `config.cfg` через `read_config()`
3. Использование значений по умолчанию при отсутствии конфига
4. Валидация и проверка конфликтов цветов
5. Вывод информации через `print_colored_system_info()`
6. Вывод информации о цветовой схеме через `print_color_scheme_info()`

**Аргументы:**
- `--default` - использование цветов по умолчанию (игнорирование конфига)

**Файл конфигурации (config.cfg):**
```
# фон для названий (левая колонка)
column1_background=4

# цвет текста для названий (левая колонка)
column1_font_color=4

# фон для значений (правая колонка)
column2_background=5

# цвет текста для значений (правая колонка)
column2_font_color=1
```

**Значения по умолчанию:**
- column1_background=1 (white)
- column1_font_color=2 (red)
- column2_background=3 (green)
- column2_font_color=4 (blue)

**Пример использования:**
```bash
./src/04/main.sh
./src/04/main.sh --default
```

### src/05/main.sh - Анализ файловой системы

**Назначение:**
Расширенный анализ директории с подсчетом файлов, папок и их типов.

**Алгоритм работы:**
1. Проверка аргументов (`--default` и путь к директории)
2. Чтение конфигурации из `config.cfg`
3. Валидация и проверка конфликтов цветов
4. Проверка пути к директории (наличие '/' в конце)
5. Проверка существования и доступности директории через `check_directory()`
6. Сбор информации через `call_with_validation_analyzer()`
7. Вывод результатов через `print_colored_analysis()`

**Аргументы:**
- `--default` - использование цветов по умолчанию
- `$1` или `$2` - путь к директории (должен заканчиваться на '/')

**Собираемая информация:**
- Общее количество папок (включая вложенные)
- Топ-5 папок по размеру
- Общее количество файлов
- Количество файлов по типам:
  - Конфигурационные (.conf)
  - Текстовые (.txt)
  - Исполняемые
  - Логи (.log)
  - Архивы
  - Символические ссылки
- Топ-10 файлов по размеру
- Топ-10 исполняемых файлов по размеру с MD5 хешами
- Время выполнения скрипта

**Пример использования:**
```bash
./src/05/main.sh /home/user/
./src/05/main.sh --default /home/user/
```

## Библиотеки (lib/)

### functions.sh - Функции сбора системной информации

**Назначение:**
Библиотека функций для сбора системной информации с валидацией.

**Функции:**

**check_dependencies()**
- Проверяет наличие необходимых команд
- Возвращает 1 при отсутствии команд
- Используемые команды: ip, grep, awk, cut, bc, find, df, hostname, timedatectl, uname, md5sum

**get_hostname()**
- Возвращает имя хоста через `hostname`

**get_timezone()**
- Возвращает часовой пояс и смещение
- Использует `timedatectl` и `date`
- Формат: "Europe/Moscow UTC +3"

**get_user()**
- Возвращает имя текущего пользователя через `whoami`

**get_os()**
- Возвращает название ОС
- Читает /etc/os-release или использует `uname -o`

**get_date()**
- Возвращает текущую дату и время
- Формат: "07 March 2026 22:41:52"

**get_uptime()**
- Возвращает время работы в читаемом формате
- Использует `uptime -p`
- Формат: "3 days, 12 hours, 47 minutes"

**get_uptime_sec()**
- Возвращает время работы в секундах
- Читает /proc/uptime

**get_ip()**
- Возвращает IP-адрес основного интерфейса
- Использует `ip route get 1`, `hostname -I` или возвращает 127.0.0.1

**get_mask()**
- Возвращает маску подсети
- Получает CIDR из `ip -o -4 addr show`
- Конвертирует CIDR в маску через массив
- Поддерживает CIDR от /0 до /32

**get_gateway()**
- Возвращает IP-адрес шлюза
- Использует `ip route | grep default`

**get_ram_total()**
- Возвращает общий объем RAM в ГБ
- Читает MemTotal из /proc/meminfo
- Преобразует КБ в ГБ через bc

**get_ram_used()**
- Возвращает использованный объем RAM в ГБ
- Вычисляет как разницу между total и free

**get_ram_free()**
- Возвращает свободный объем RAM в ГБ
- Читает MemFree из /proc/meminfo

**get_space_root()**
- Возвращает общий объем корневого раздела в МБ
- Использует `df /`
- Преобразует КБ в МБ

**get_space_root_used()**
- Возвращает использованный объем корневого раздела в МБ

**get_space_root_free()**
- Возвращает свободный объем корневого раздела в МБ

**prompt_yes_no()**
- Запрос подтверждения у пользователя
- Параметр: сообщение для отображения
- Возвращает 0 для Y/y, 1 для N/n

**call_with_validation()**
- Универсальная обертка для безопасного вызова функций
- Параметр: имя функции
- Проверяет наличие функции, ее результат и валидирует специфические типы данных
- Завершает скрипт при ошибке

**call_with_validation_analyzer()**
- Обертка для безопасного вызова функций анализа файловой системы
- Параметры: имя функции, аргумент функции
- Проверяет наличие функции и валидирует числовые результаты
- Завершает скрипт при ошибке

**print_system_info()**
- Единообразный вывод информации
- Параметры: метка, значение
- Форматирует вывод с выравниванием метки до 20 символов

### colors.sh - Функции цветного вывода

**Назначение:**
Библиотека функций для цветного вывода с ANSI-кодами.

**Функции:**

**get_text()**
- Возвращает ANSI-код цвета текста по номеру (1-6)
- Параметр: номер цвета
- Возвращает: TEXT_WHITE, TEXT_RED, TEXT_GREEN, TEXT_BLUE, TEXT_PURPLE, TEXT_BLACK

**get_bg()**
- Возвращает ANSI-код цвета фона по номеру (1-6)
- Параметр: номер цвета
- Возвращает: BG_WHITE, BG_RED, BG_GREEN, BG_BLUE, BG_PURPLE, BG_BLACK

**get_color_name()**
- Возвращает название цвета по номеру
- Параметр: номер цвета
- Возвращает: white, red, green, blue, purple, black

**print_colored_system_info()**
- Выводит системную информацию с цветным форматированием
- Параметры: метка, значение
- Автоматически форматирует метку до 20 символов
- Использует цвета из переменных bg_names, text_names, bg_values, text_values

**print_colored_analysis()**
- Выводит информацию анализа с цветным форматированием
- Параметры: метка, значение
- Автоматически форматирует метку до 35 символов
- Использует цвета из переменных bg_names, text_names, bg_values, text_values

**print_color_scheme_info()**
- Выводит информацию о текущей цветовой схеме
- Параметр: источник конфигурации ("default" или "config")
- Показывает используемые цвета для всех элементов

### config.sh - Функции работы с конфигурацией

**Назначение:**
Библиотека функций для чтения конфигурации и валидации цветов.

**Функции:**

**read_config()**
- Читает конфигурационный файл в ассоциативный массив
- Параметры: путь к файлу, имя ассоциативного массива (по ссылке)
- Игнорирует комментарии и пустые строки
- Возвращает 0 при успехе, 1 при ошибке

**validate_color_value()**
- Проверяет, что значение цвета — число от 1 до 6
- Параметры: значение, название параметра
- Выводит ошибку и возвращает 1 при невалидном значении

**validate_all_colors()**
- Проверяет все значения цветов
- Параметры: bg_names, text_names, bg_values, text_values
- Использует validate_color_value() для каждого параметра
- Возвращает 1 при любой ошибке

**check_color_conflict()**
- Проверяет, что цвет текста и фона не совпадают
- Параметры: bg_color, text_color, column_name
- Выводит ошибку и возвращает 1 при совпадении

**check_all_color_conflict()**
- Проверяет конфликты цветов для всех столбцов
- Использует check_color_conflict() для названий и значений
- Возвращает 1 при любом конфликте

### analyzer.sh - Функции анализа файловой системы

**Назначение:**
Библиотека функций для анализа файловой системы.

**Функции:**

**count_folders()**
- Считает общее число папок (включая вложенные)
- Параметр: путь к директории
- Использует `find` и `wc -l`

**top_folders()**
- Выводит топ-N папок по размеру
- Параметры: путь к директории, количество папок
- Использует `du`, `sort`, `head`
- Форматирует размер через `format_size()`

**count_files()**
- Считает общее число файлов
- Параметр: путь к директории
- Использует `find` и `wc -l`

**count_files_by_ext()**
- Считает число файлов с заданным расширением
- Параметры: путь к директории, расширение
- Использует `find` и `wc -l`

**count_executable()**
- Считает число исполняемых файлов
- Параметр: путь к директории
- Использует `find -executable` и `wc -l`

**count_archive()**
- Считает число архивных файлов
- Параметр: путь к директории
- Ищет файлы с расширениями: .tar, .gz, .zip, .7z, .rar

**count_symlinks()**
- Считает число символических ссылок
- Параметр: путь к директории
- Использует `find -type l` и `wc -l`

**format_size()**
- Форматирует размер файла в человекочитаемый вид
- Параметр: размер в байтах
- Возвращает: B, KB, MB, GB
- Использует `bc` для деления с плавающей точкой

**top_files()**
- Выводит топ-N файлов по размеру
- Параметры: путь к директории, количество файлов
- Использует `find`, `du`, `sort`, `head`
- Показывает путь, размер и расширение

**top_executable()**
- Выводит топ-N исполняемых файлов по размеру
- Параметры: путь к директории, количество файлов
- Использует `find`, `du`, `sort`, `head`
- Показывает путь, размер и MD5 хеш

**check_directory()**
- Проверяет директорию на существование, тип и права
- Параметр: путь к директории
- Проверяет:
  - Не пустой путь
  - Заканчивается на '/'
  - Существует
  - Является директорией (не файлом)
  - Доступен для чтения
- Возвращает 1 при любой ошибке
