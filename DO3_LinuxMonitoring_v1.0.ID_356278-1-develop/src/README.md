# Linux Monitoring v1.0 - src/

Директория с исходным кодом проекта Linux Monitoring.

## Структура проекта

```
src/
├── 01/               # Часть 1: Базовый скрипт
├── 02/               # Часть 2: Сбор системной информации
├── 03/               # Часть 3: Цветовой вывод
├── 04/               # Часть 4: Конфигурация цветов
├── 05/               # Часть 5: Анализ файловой системы
└── lib/              # Библиотеки функций
    ├── functions.sh  # Функции сбора системной информации
    ├── colors.sh     # Функции цветного вывода
    ├── config.sh     # Функции работы с конфигурацией
    └── analyzer.sh   # Функции анализа файловой системы
```

## Описание компонентов

### src/01 - Первое усилие

Простой скрипт, проверяющий аргумент командной строки. Выводит значение параметра, если оно не является числом.

### src/02 - Исследование системы

Скрипт сбора системной информации:
- Имя хоста, часовой пояс, текущий пользователь
- Информация об ОС, текущая дата и время
- Время работы системы (в секундах и читаемом формате)
- Сетевая информация (IP, маска, шлюз)
- Информация о памяти (RAM)
- Информация о дисковом пространстве
- Возможность сохранения результатов в файл

### src/03 - Визуальное оформление

Добавлен цветной вывод с параметрами командной строки для настройки цветовой схемы:
- 4 параметра для настройки цветов (фон и текст для названий и значений)
- Валидация цветов (недопустимо совпадение цвета текста и фона)

### src/04 - Конфигурация визуального оформления

Реализовано чтение конфигурации из файла config.cfg:
- Возможность использования значений по умолчанию
- Поддержка параметра --default для принудительного использования цветов по умолчанию
- Вывод информации о текущей цветовой схеме

### src/05 - Исследование файловой системы

Расширенный анализ директории:
- Подсчет количества папок и файлов
- Анализ типов файлов (конфигурационные, текстовые, исполняемые, логи, архивы, символические ссылки)
- Топ-N папок и файлов по размеру
- Топ-N исполняемых файлов с хешами
- Время выполнения скрипта

## Библиотеки (lib/)

### functions.sh

Функции для сбора системной информации:
- get_hostname() - имя хоста
- get_timezone() - часовой пояс
- get_user() - текущий пользователь
- get_os() - информация об ОС
- get_date() - текущая дата и время
- get_uptime() - время работы (читаемый формат)
- get_uptime_sec() - время работы (в секундах)
- get_ip() - IP-адрес
- get_mask() - маска подсети
- get_gateway() - шлюз
- get_ram_*() - информация о памяти
- get_space_root_*() - информация о дисковом пространстве
- call_with_validation() — универсальная обертка для безопасного вызова функций
- call_with_validation_analyzer() — обертка для безопасного вызова функций анализа файловой системы
- print_system_info() - единообразный вывод информации

### colors.sh

Функции для цветного вывода:
- get_text() - ANSI-код цвета текста
- get_bg() - ANSI-код цвета фона
- get_color_name() - название цвета по номеру
- print_colored_system_info() - вывод системной информации с цветным форматированием
- print_colored_analysis() - вывод информации анализа с цветным форматированием
- print_color_scheme_info() - вывод информации о цветовой схеме

### config.sh

Функции для работы с конфигурацией:
- read_config() - чтение конфигурационного файла
- validate_color_value() - валидация значения цвета
- validate_all_colors() - валидация всех цветов
- check_color_conflict() - проверка конфликта цветов
- check_all_color_conflict() - проверка конфликтов для всех столбцов

### analyzer.sh

Функции для анализа файловой системы:
- count_folders() - подсчет папок
- top_folders() - топ папок по размеру
- count_files() - подсчет файлов
- count_files_by_ext() - подсчет файлов по расширению
- count_executable() - подсчет исполняемых файлов
- count_archive() - подсчет архивов
- count_symlinks() - подсчет символических ссылок
- format_size() - форматирование размера
- top_files() - топ файлов по размеру
- top_executable() - топ исполняемых файлов
- check_directory() - проверка директории

## Требования

- Bash 4.0+
- Ubuntu Server 20.04 LTS (рекомендуемая среда выполнения)
- Утилиты: bc, find, du, df, ip, grep, awk, md5sum

## Использование

```bash
# Часть 1
./src/01/main.sh "test"

# Часть 2
./src/02/main.sh

# Часть 3
./src/03/main.sh 1 2 3 4

# Часть 4
./src/04/main.sh [--default]

# Часть 5
./src/05/main.sh /path/to/directory/
./src/05/main.sh --default /path/to/directory/
```
