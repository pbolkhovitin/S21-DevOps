## Part 2. Создание пользователя

**Задача:** Создать пользователя, отличного от созданного при установке, и добавить его в группу `adm`.

**Выполнение:**
```bash
# Создание пользователя:
sudo adduser do1-new_user

# Добавление в группу adm
sudo usermod -aG adm do1-new_user

# Проверка нового пользователя: 
cat /etc/passwd | grep do1-new_user
```

Пояснение использованных команд и ключей:
- `adduser` - интерактивная утилита создания пользователей (автоматически создает домашнюю директорию, устанавливает пароль)
- `usermod` - утилита модификации учетных записей пользователей
- `-aG` - ключи команды usermod:
  - `-a` (append) - добавить пользователя к дополнительным группам без удаления из текущих
  - `-G` (groups) - указать список дополнительных групп
- `adm` - группа, предоставляющая доступ к чтению системных логов

**Просмотр сетевых интерфейсов**
- lo: `state UNKNOWN` - нормально для loopback
- ens18: `state UP` - интерфейс активен и готов к работе
- Оба интерфейса в состоянии `UP, LOWER_UP` - корректно работают

![Сетевые интерфейсы](media/part3_network_interfaces.png)

>Интерфейс **lo (loopback)** - виртуальный сетевой интерфейс для локальной коммуникации внутри системы.<br>
>Используется для тестирования сетевых приложений без реального сетевого подключения.<br>
>Имеет фиксированный IP-адрес 127.0.0.1

**Получени адреса от DHCP**

```bash
sudo dhclient -r ens18  # Освобождение текущего IP
sudo dhclient -v ens18  # Получение нового IP с подробным выводом
ip addr show            # Проверка результата
```

![IP от DHCP](media/part3_dhcp_ip.png)

>**DHCP (Dynamic Host Configuration Protocol)** - протокол динамической конфигурации узлов.<br>
>Автоматически назначает IP-адреса и сетевые параметры (шлюз, DNS-серверы).<br>
>Позволяет подключать устройства к сети без ручной настройки

```
Расшифровка DHCP:
Процесс работы:
DISCOVER - клиент ищет DHCP сервер
OFFER - сервер предлагает IP адрес
REQUEST - клиент запрашивает предложенный IP
ACK - сервер подтверждает выделение IP
Результат:
Успешно получен IP: 192.168.101.164/24 от DHCP сервера 192.168.101.100
Статус: dynamic - подтверждает получение от DHCP
Аренда: 43177 секунд (~12 часов)
Получен IP: 192.168.101.164/24
```
**Определение шлюза**

![Статические настройки](media/part3_route_dhcp.png)

- Маршрут по умолчанию: `default via 192.168.101.100 dev ens18` через интерфейс ens18
- Локальная сеть: `192.168.101.0/24` с исходным адресом `src 192.168.101.164`
- Внешний IP шлюза: 192.168.101.100

**Выполнение:**

```bash

```
**Результаты:**

![Установка sshd](media/part8_sshd_install.png)
![Вывод sshd](media/part8_sshd_process.png)

**Команда `ps aux | grep sshd`:**

- `ps` - показывает запущенные процессы
- `aux` - ключи: `a` (все процессы), `u` (пользовательский формат), `x` (процессы без терминала)

![Вывод команды top](media/part8_netstat_output.png)

>**Команда `netstat -tan`:**
- -t - отображать только TCP соединения
- -a - показать все соединения (и прослушивающие, и установленные)
- -n - выводить адреса и порты в числовом формате (без разрешения имен)

**Столбцы:**
- Proto - протокол (tcp/tcp6)
- Recv-Q - количество байт, не скопированных программой пользователя
- Send-Q - количество байт, не подтвержденных удаленным хостом
- Local Address - локальный адрес и порт
- Foreign Address - удаленный адрес и порт
- State - состояние соединения

LISTEN - состояние ожидания входящих подключений
-  `tcp 0 0 0.0.0.0:2022 0.0.0.0:* LISTEN` ✓
- `0.0.0.0:2022` - служба слушает на всех интерфейсах порт 2022

**Вывод:** SSH сервер успешно установлен и работает на порту 2022.
